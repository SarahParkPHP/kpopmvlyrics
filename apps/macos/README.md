# K-Pop MV Lyrics ‚Äî macOS (SwiftUI)

Native macOS frontend. It contains **no business logic**: every operation goes
to the shared Rust core (`src-tauri`) through a UniFFI-generated boundary, the
same `AppContext` the GTK4/Linux UI uses in-process. Data crosses through the
shared JSON command surface; the GStreamer video renders directly into an
`NSView` via the core's player thread.

## Prerequisites

- **Xcode 15+** (macOS 13+ deployment target).
- **Rust** (stable) with the Apple targets:
  `rustup target add aarch64-apple-darwin x86_64-apple-darwin`
- **GStreamer** development + runtime for macOS (the official
  `GStreamer.framework` under `/Library/Frameworks`). The core links it via
  pkg-config when building, and the app links `-framework GStreamer`.
- **XcodeGen** (`brew install xcodegen`) to create the project from `project.yml`.
- Optional: `swiftformat` (UniFFI auto-formats generated Swift if present).

## Build & run

```sh
cd apps/macos
./Scripts/build-rust.sh debug     # build universal static lib + regenerate bindings
xcodegen generate                 # produce Kpml.xcodeproj
open Kpml.xcodeproj               # build & run from Xcode (‚åòR)
```

The Xcode project also runs `Scripts/build-rust.sh` as a pre-build step, so after
the first setup you can just build from Xcode and the Rust core/bindings stay in
sync. `build-rust.sh release` produces an optimized build.

## How the boundary works

- `CoreClient` wraps the generated `Core` object. `invoke(name, argsJSON)` covers
  the command surface; `playerLoad/Play/Pause/Seek/AttachSurface/Snapshot` drive
  playback; `setObserver` registers a `PlaybackObserver` that the core calls with
  `video-position` / `video-player-error` events (delivered to the main actor).
- `VideoSurfaceView` (`NSViewRepresentable`) passes its `NSView` pointer to
  `playerAttachSurface`; GStreamer's `osxvideosink` renders into it.
- JSON export is built by the **Rust core** (`build_export` command), shared with
  the GTK4 editor ‚Äî one canonical shape across all frontends.

## Status / caveats

- The Rust core, UniFFI surface, and bindings are verified to build and
  generate. **This SwiftUI app has not yet been compiled in Xcode** (developed on
  Linux), so expect minor Swift adjustments on first build.
- The player runs GStreamer on a dedicated GLib thread. `osxvideosink` + `NSView`
  is AppKit-main-thread-sensitive; **video rendering needs on-device validation**.
- For distribution the GStreamer dylibs/framework must be embedded in the app
  bundle and code-signed (not covered here ‚Äî dev setup assumes a system
  `/Library/Frameworks/GStreamer.framework`).
- The app is currently non-sandboxed (dev). Open/Save panels grant file access;
  add an entitlements file before shipping.
